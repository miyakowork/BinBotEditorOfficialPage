# BinBotEditor 中文官方网站设计规格

## 目标

为 BinBotEditor 建立一个面向中文用户的 macOS 产品官网。首版的首要转化目标是下载，但产品尚未正式发布，因此所有下载入口统一显示“macOS 版即将发布”，不提供虚假下载地址，也不收集用户信息。

网站参考 Open Design 官网的大字号产品叙事、真实产品界面展示和章节式长页面节奏，但不复制其品牌资产、文案或具体页面。BinBotEditor 使用自己的图标、深蓝紫品牌色和编辑器产品界面。

## 受众与成功标准

核心受众是希望获得轻量、专注、本地优先代码编辑体验的 macOS 开发者。

首版成功标准：

- 用户在首屏内理解 BinBotEditor 是一款 macOS 代码编辑器。
- 用户能快速识别产品的专注编辑、多标签、语言支持、内置格式化和本地优先特性。
- 所有“下载”入口明确表达尚未发布，行为一致且可访问。
- 页面在桌面、平板和手机上均完整可用，并尊重减少动态效果的系统设置。
- 项目可通过静态构建部署，不依赖后端或第三方运行时服务。

## 视觉方向

采用已确认的 B 方案“夜幕工作台”。页面以近黑和深蓝紫为主，局部使用 BinBotEditor 图标中的淡紫高光与半透明玻璃层次。整体气质沉浸、克制、原生，避免常见的紫色渐变卡片堆砌。

标题使用大字号、紧凑字距和明显层级；正文保持舒适行宽与充足留白。背景可使用细微网格、柔光和低对比噪点增强空间感，但不得影响文字对比度。产品窗口是主要视觉锚点，使用原创编辑器界面或真实产品截图，不依赖远程图片。

## 信息架构

页面采用中文单页结构：

1. 固定透明导航：品牌、功能、格式化、隐私、即将发布。
2. 首屏：主标题“思考，在代码之前。”、简短产品定位和“macOS 版即将发布”主按钮。
3. 产品窗口：大幅展示编辑器工作区，并随滚动产生克制的上浮效果。
4. 核心价值：专注编辑、多标签工作区、原生 macOS 体验。
5. 语言与格式化：展示已支持的语言及完全内置、离线可用的格式化能力。
6. 本地与隐私：说明文件和编辑流程留在用户设备上，不暗示尚未实现的云能力。
7. 发布预告：再次呈现 macOS 即将发布状态。
8. FAQ：回答平台、发布时间、数据位置、格式化与价格等高频问题；未知发布时间明确标注尚未公布。
9. 页脚：品牌、版本状态和必要链接。

## 技术架构

使用 React、TypeScript 和 Vite 构建静态单页。该技术栈与 BinBotEditor 前端保持一致，便于后续复用品牌组件和维护方式。首版不接后端、不使用数据库、不收集邮箱或分析数据。

页面拆分为独立组件：导航、首屏、产品窗口、核心功能、语言与格式化、本地隐私、FAQ、发布 CTA 和页脚。页面内容保存在本地静态数据中，FAQ 展开状态和移动导航状态仅存在浏览器内存，不持久化。

## 交互与状态

- 导航链接滚动到对应章节，并提供清晰的键盘焦点样式。
- 移动端导航使用可通过键盘操作的折叠菜单，打开后管理正确的 `aria-expanded` 状态。
- “即将发布”按钮触发一致的轻量提示，说明当前没有可下载文件；不会跳转到空地址。
- FAQ 使用原生或等价语义的可访问折叠结构。
- 动画使用 CSS 和少量 Intersection Observer：首屏分层入场、章节显现、产品窗口轻微视差。
- 当系统启用 `prefers-reduced-motion` 时，禁用非必要动画和视差。
- 移动端减少背景特效与动画强度，优先保证性能和可读性。

## 内容边界

首版仅介绍 macOS 产品并使用中文。不包含 Windows/Linux 下载、多语言切换、账号系统、订阅表单、博客、在线编辑器、真实下载文件、支付、遥测或云端服务。未来增加真实发布地址时，只替换统一下载配置与状态，不改动各章节结构。

## 错误与降级

静态内容无网络依赖。JavaScript 未执行时，主要文案与章节仍可阅读；菜单和 FAQ 应尽可能使用渐进增强方式。图标或装饰资源加载失败时不得阻断内容。未知发布信息统一显示“即将发布”，不得展示猜测日期。

## 响应式与无障碍

重点验证约 1440px 桌面、768px 平板和 390px 手机视口。使用语义化标题结构、导航、按钮和页脚；保证文本对比度、键盘可操作性、可见焦点、合理触控目标，并为装饰图形隐藏无意义的辅助技术描述。

## 验证策略

- TypeScript 类型检查和 Vite 生产构建。
- 组件或 DOM 测试覆盖移动导航、FAQ、统一的“即将发布”行为和关键文案。
- 桌面、平板、手机截图检查布局、溢出和视觉层级。
- 键盘导航与减少动态效果检查。
- 确认生产构建不依赖远程图片或后端服务。

## 交付与仓库

本地工作目录为 `BinBotEditorOfficialHomepage`。GitHub 上创建用户指定的私有仓库 `BinBotEditorOfficialPage`，默认分支为 `main`。完成实现和验证后，将本地提交推送到该私有仓库。
