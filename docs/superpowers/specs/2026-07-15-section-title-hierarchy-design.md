# BinBotEditor 官网 Section 标题层级精修设计

## 1. 问题与目标

当前首屏的超大标题已经承担品牌视觉高潮，但后续产品能力、工作流、Mac、FAQ 与下载区域继续重复接近首屏的字号和强制断行，造成章节之间缺少强弱关系。部分标题在窄内容列中被拆成三行，正文又通过负边距向上追赶，视觉上显得突兀且拥挤。

本轮采用用户确认的 B 方案“编辑式层级收敛”：保留首屏的大标题表现力，后续标题改为更稳定、克制的编辑排版。信息架构、文案含义、功能演示、配色与下载逻辑均不改变。

## 2. 排版层级

### 2.1 章节主标题

- 产品能力、工作流、Mac、FAQ 与下载区主标题统一使用自然断行，不在组件中插入固定 `<br />`。
- 桌面字号控制在 `52–72px`，行高约 `1.06–1.12`，字距由当前约 `-0.065em` 收敛到约 `-0.045em`。
- 平板字号控制在 `48–62px`，手机字号控制在 `38–48px`。
- 标题不再与首屏争夺最大视觉权重；眉题、标题和正文通过间距形成层级，不依赖夸张字号。

### 2.2 内容标题

- 四个产品能力标题由当前最高约 `70px` 收敛到 `38–54px`，行高提高到约 `1.08`。
- 工作流列表、FAQ 问题与引用文案保持现有层级，仅调整它们与主标题之间的垂直节奏。
- 数字 `30+`、`14`、`MD` 和 `{}` 继续作为功能视觉数据，不与正文标题共用尺寸规则。

## 3. 构图与留白

- 产品能力和工作流的 `.section-heading` 保留“小眉题 + 主标题”结构，桌面使用窄左列和宽右列；主标题改为左对齐并限制最大宽度，不再使用大面积右对齐块。
- 章节标题到首个内容分隔线的距离由当前约 `70px` 收敛到 `48–56px`。
- Mac 区域标题控制在两行以内，并移除正文向上挤压的负边距。
- FAQ 标题保持单行或自然换行，减少标题下方空白。
- 下载区标题仍可比普通章节略大，但上限低于首屏标题，并限制文本宽度形成稳定居中构图。

## 4. 响应式与动效

- 980px 以下取消章节标题的双列错位，眉题位于标题上方，全部左对齐。
- 760px 以下使用单列自然换行，避免孤字、三行短句和横向溢出。
- 延续现有 Reveal 入场，不新增逐字动画、持续漂移或大幅缩放；标题收敛后通过留白和阅读顺序建立节奏。
- `prefers-reduced-motion`、触控降级和键盘焦点行为保持不变。

## 5. 组件与样式范围

- `CapabilitySections.tsx`、`WorkflowStrip.tsx`、`MacSection.tsx`：移除标题中的固定换行。
- `styles.css`：拆分章节主标题、能力标题、Mac 标题、FAQ 标题和下载标题的尺寸与间距规则，避免一个全局规则控制所有场景。
- 不修改 `Hero` 的首屏标题，不修改产品文案、能力数字、下载链接或发布状态。

## 6. 验证标准

- 自动测试验证 Section 标题不再依赖 `<br />`，并检查桌面与手机字号上限、行高和对齐契约。
- 以 1440 × 1024、768 × 1024 和 390 × 844 浏览器视口检查标题断行、留白、正文衔接和横向溢出。
- 对比修改前后的产品能力、工作流、Mac、FAQ 与下载区截图；首屏视觉权重应明显高于后续章节。
- 全量 Vitest、TypeScript 编译、Vite 生产构建和 `git diff --check` 必须通过。

## 7. 非目标

本轮不重写文案，不改变 Section 顺序，不新增插画或字体，不调整产品演示状态，不修改 GitHub Pages 架构。
